# WorkHelper

## はじめて使う方（3ステップ）

1. ブラウザで **[Releases ページ](https://github.com/Akeytec/WorkHelper/releases)** を開きます。  
2. 一覧の**最新**のところの **Assets** から、**`WorkHelper Setup x.x.x.exe`**（例: `WorkHelper Setup 2.0.0.exe`）をダウンロードし、保存したファイルを**ダブルクリック**してインストールします。  
3. 完了したら、**スタートメニュー**（または「すべてのアプリ」）の **WorkHelper** を起動します。  

> 開発者向け（ソースから動かす・ビルドする）の手順は **[`docs/開発者向け.md`](docs/開発者向け.md)** だけを参照してください。`git` や `npm` の知識は、**この README を読む利用者**には不要です。

---

## 概要

WorkHelper は **Windows 向けのデスクトップアプリ**（インストーラーで導入）です。**オフライン**で、タスク・メモ・社員・時間外などの作業補助に使います。データは PC 上（アプリの保存領域、または指定したフォルダ）に保存します。

- **導入**: 配布の **Setup（`.exe`）** を実行してインストールするだけで利用できます。別途 **Node.js** や **ブラウザの手動起動**は不要です。  
- **更新**: 配布元が **GitHub Releases** などで公開している新バージョンが出た場合、起動中に更新の案内が表示されるか、**ヘルプ**メニューから**更新を確認**できます（ネットワーク接続と、アプリ配布の都合が必要です）。まずは **README や社内手順**に従ってください。

## インストール（補足）

- **上記「はじめて使う方」**のとおり **Releases から**入手する方法が、公式の取得方法です。  
- **社内でインストーラを直接渡してもらう**場合は、その `.exe` をダブルクリックして同様に導入できます。  
- インストール中に**インストール先フォルダ**を選べる画面が出たら、分かる範囲で指定してかまいません（分からなければ**既定**のままで問題ありません）。  

> **補足:** ダウンロードした `.exe` を右クリック → **プロパティ** → **「ブロックの解除」**にチェックが付いている場合は、外してから実行してください。

## 初回に表示されることがある警告

- **Windows の保護**（SmartScreen など）  
  社内配布と**コード署名**の有無によって、**いったん止める**表示が出ることがあります。**詳細**から内容を確認し、配布元が信頼できる場合に実行を選んでください。  
- **スマート アプリ コントロール**（Windows 11）  
  厳格な環境では、**「実行」がすぐに選べない**場合があります。社内の**情報システム**の案内、または**設定**で対応が必要なことがあります。  
- **配布者側**でできる根本的な緩和のひとつは、インストーラーに**デジタル署名**を付けることです。詳細は **[`docs/開発者向け.md`](docs/開発者向け.md)** を参照してください。

## データの保存

- **通常**  
  タスク・メモ・社員・時間外の設定などは、**PC 上のアプリ専用フォルダ**（ユーザーごと）に保存されます。  
- **「フォルダを選ぶ」**（バックアップ画面）  
  保存先のフォルダを指定すると、**`workhelper-storage.json`** にも同じ系統の内容を書き出し、手元のバックアップや別 PC への持ち出しに使えます。  
- **別 PC へ**  
  フォルダを丸ごとコピーするだけではデータは引き継がれません。**バックアップのエクスポート（JSON）**の利用を推奨します。

## 更新（GitHub 配布の場合の例）

1. 配布元が新しい **Setup または自動更新**を案内しているか確認する。  
2. **自動更新**が有効なビルドでは、利用可能な更新のダウンロードが終わったあと、**再起動**の案内が出る場合があります。  
3. 手動で入れ替える場合は、**新しいインストーラー**を実行し、**更新前**に **バックアップ画面**で **「すべてエクスポート（.json）」** しておくと安全です。  

※ 手順の詳細は **[`更新のしかた.txt`](更新のしかた.txt)** および **社内の案内**に従ってください。

## うまく動かないとき

| 症状 | 確認すること |
|------|----------------|
| インストールが途中で止まる | 管理者権限が必要な場合、**右クリック → 管理者として実行**。ウイルス対策で止められていないか。 |
| 起動しても画面が出ない | 一度ログオフ/再起動。タスクマネージャーで**二重起動**していないか。 |
| 更新を確認できない | **インターネット**に接続しているか。社内の**プロキシ / セキュリティ**で GitHub へのアクセスが許可されているか。 |

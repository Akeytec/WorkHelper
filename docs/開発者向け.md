# WorkHelper — 開発者・配布担当者向け

利用者向けの説明は [**`README.md`**](../README.md) を参照してください。

## 技術スタック

- **Electron**（メインプロセス: [`electron/main.cjs`](../electron/main.cjs)、プリロード: [`electron/preload.cjs`](../electron/preload.cjs)）
- 画面は [`app/`](../app/) の HTML / CSS / JavaScript（ビルドステップなしで asar に同梱）
- **永続化**: `app.getPath("userData")` 内の `app-state.v1.json` ＋ 任意の連携フォルダ内 `workhelper-storage.json`（[`electron/main.cjs`](../electron/main.cjs) で集約）
- **更新**: [`electron-updater`](https://www.electron.build/auto-update) + **GitHub Releases**（[`package.json`](../package.json) の `build.publish`）

## ローカル開発

```bat
cd WorkHelper
npm install
npm start
```

開発時は `electron-updater` は動作を抑止（`app.isPackaged` が false）。

## インストーラーのビルド（Windows）

```bat
npm run dist
```

成果物は **`dist/`**（`.gitignore` 済み）。例: `WorkHelper-Setup-2.0.5.exe`、検証用は `dist\win-unpacked\WorkHelper.exe`。

- **アイコン**: 未設定の場合は Electron の既定アイコン。差し替えるには `build/icon.ico` を置き、[`package.json`](../package.json) の `build.win.icon` を指定（[electron-builder](https://www.electron.build/) 参照）。
- **コード署名**: 本番配布では Authenticode 署名を推奨（未署名だと SmartScreen / スマートアプリ コントロールで止まりやすい）。`signtool` 等は OS ・証明書の手配に依存する。

## 初回: GitHub に空のリポジトリを作り、ローカルから接続する

利用者が **Releases からインストーラを取れる**ようにするには、**Public** リポジトリを推奨します（[README](../README.md) の「はじめて使う方」は `Akeytec/WorkHelper` を前提にしています）。

1. GitHub で **New repository** → Owner **`Akeytec`**, 名前 **`WorkHelper`**, **Public** → README 等は追加せず **Create**（中身は手元からプッシュする）。  
2. リポジトリのローカルで（初回のみ）:

   ```bat
   cd c:\WorkHelper
   git init
   git add .
   git commit -m "Initial commit: WorkHelper 2.0 Electron"
   git branch -M main
   git remote add origin https://github.com/Akeytec/WorkHelper.git
   git push -u origin main
   ```

3. 初回 `git push` では **Git 認証**（GitHub ログイン、Git Credential Manager、または Personal Access Token）が必要な場合があります。**手順 1 のリポジトリを先に作っておかない**と `git push` は失敗します。コマンドが認証待ちで止まる場合は、ターミナルまたはポップアップの指示に従って完了してください。

## Internal self-signed release flow

Because Intune, Group Policy, and Active Directory are not available, each company PC must trust the WorkHelper certificate once before running the signed installer.

### First time: create the certificate on the distributor PC

PowerShell command:

    powershell -NoProfile -ExecutionPolicy Bypass -File tools\signing\create-self-signed-code-signing-cert.ps1

The output directory is signing-local/. WorkHelper-CodeSigning.pfx contains the private key and must never be committed. WorkHelper-CodeSigning.cer is the public certificate installed on user PCs.

### First time on each user PC

Download WorkHelper-Certificate-Setup.zip from the signed Release assets, extract it, and run the certificate registration BAT file. This imports the public certificate into the CurrentUser Root and TrustedPublisher stores.

### Create a signed Release

Commit and push all source changes first, then run the local signed release script with a GitHub token.

PowerShell command:

    $env:GH_TOKEN = your_github_token
    powershell -NoProfile -ExecutionPolicy Bypass -File tools\signing\release-signed.ps1

The script signs the Electron installer, creates and pushes the vX.Y.Z tag, and uploads the installer, blockmap, latest.yml, certificate setup ZIP, public certificate, and certificate registration scripts to GitHub Releases.

If the certificate is recreated, every user PC must install the new public certificate again.

## GitHub Releases

Do not use tag-push automatic releases, because they would publish unsigned installers. Signed releases are created locally with tools/signing/release-signed.ps1.

1. Update package.json version and app/app.js APP_VERSION.
2. Commit and push the changes, then make sure the working tree is clean.
3. Set GH_TOKEN and run tools/signing/release-signed.ps1.
4. Confirm that GitHub Releases contains the signed installer, latest.yml, and WorkHelper-Certificate-Setup.zip.

For Private repositories, auto-update requires authentication for user PCs. For this project, Public Releases or a separate internal distribution route are simpler.

## ExcelJS（`app/vendor/exceljs.min.js`）

時間外一覧の Excel 出力用。更新手順:

1. `npm install exceljs@<version>`  
2. `node_modules/exceljs/dist/exceljs.min.js` を `app/vendor/exceljs.min.js` にコピー  
3. `app/vendor/NOTICE.txt` を確認
